# Roadmap to PI Planning Readiness Workflow

## Trigger

- Prepare this for PI planning
- Check PI readiness

## Required Input

- roadmap item
- epic or feature context
- capacity
- dependencies
- architecture readiness
- data readiness

## Steps

1. Break roadmap item into PI-sized scope options.
2. Check architecture and data readiness gaps.
3. Validate dependency readiness and external blockers.
4. Highlight capacity tradeoffs and sequencing.
5. Name required decisions before commit.

## Output

- feature breakdown
- readiness gaps
- PI candidate scope
- risks
- open questions
- required decisions

## Save Output

- Artifacts/roadmaps/YYYY-MM-DD-pi-readiness-topic.md
- Use Templates/pi-planning-readiness.md
