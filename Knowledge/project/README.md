# Project Knowledge

Use this folder for local references when handling Jira + Confluence epic/feature work.

Purpose:
- Keep reusable local context for future sessions.
- Track what was created/updated in Jira and Confluence.

Recommended structure:
- `Knowledge/project/<initiative>/<jira-key>.source.md`
- `Knowledge/project/<initiative>/<jira-key>.notes.md`
- Optional diagram source: `Knowledge/project/<initiative>/<jira-key>.source.puml`

Minimum content to capture per feature:
- Jira key and title
- Confluence page link
- In-scope and out-of-scope summary
- Key assumptions and decisions
- Open questions and follow-up actions
