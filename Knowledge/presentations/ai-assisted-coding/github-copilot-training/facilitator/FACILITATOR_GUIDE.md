# FACILITATOR_GUIDE.md

## Overview

Full-day training (~6h) for energy data engineers and data scientists.
Uses a real incident in a market data pipeline as the teaching story.

Central narrative:
> A CET copy-through bug was invisible in winter data for 10 months.
> Only a summer regression caught it: 29,211 mismatches.
> Fixed to 6 (order B edge case). Fixed to 0.
> The regression runner is what separates "it works in February" from "it works in June."

## Pre-training setup

### What you prepare before the day

**Data files** (generated by `python data/generate_sample_data.py` — already in repo):
- `data/curve_winter.csv` — winter data, no CET issues
- `data/curve_march.csv` — spring forward data
- `data/curve_summer.csv` — summer data; 75 mismatches with copy-through bug
- `data/curve_october.csv` — last Sunday October, order A + B
- `regression/checkpoints/checkpoint_winter.csv`
- `regression/checkpoints/checkpoint_march.csv`
- `regression/checkpoints/checkpoint_summer.csv`
- `regression/checkpoints/checkpoint_october.csv`

### Day-of checks

Before participants arrive:
1. `pytest tests/test_curve_loader.py` → green
2. Regression against `checkpoint_winter.csv` → 0 mismatches
3. Regression against `checkpoint_summer.csv` → 75 mismatches (confirms bug is present)
4. `cet_converter.py` on `main` is still the buggy copy-through version

## Key state invariants

These must hold throughout the training or modules break:

| Invariant | Why it matters |
|---|---|
| `cet_converter.py` on `main` stays buggy through M2 and M3 | M4 TDD depends on the bug being present to write meaningful tests |
| M4 ends with green `pytest tests/` and 0 regressions on all seasons | M5 imports `CurveValidator` and fixed `cet_converter` |
| `.copilotignore` must not be removed | Protects M2–M6 exercises from context leaking through facilitator files |

---

## Common failure modes (quick reference)

| Symptom | Likely cause | Response |
|---|---|---|
| Copilot fixes summer but breaks order B | Missing order B test | "Run `/review-cet-function` — which regime is missing?" |
| Winter regression passes but summer fails | Expected — this is the reveal | Let it happen |
| SODA checks not surfacing issues | Wrong column name | "Run against `curve_with_issues.csv` and check row 2" |
| M6 dashboard has wrong values | Agent invented data instead of reading CSVs | "How do you know the values match the CSV?" |
| Copilot solves the bug in M2 Step 1 | Context leaked from other open files | Open a fresh chat, close other notebooks |

## Closing debrief (15 min after M6)

Ask the room: "We started with 29,211 mismatches that lived undetected for 10 months. Walk me through every module and name the thing you built or learned that would have caught it earlier."

Go around the room. Answers are the day's takeaways.

Expected answers:
- M1: understanding the implicit assumptions in `curve_loader.py`
- M2: `/fix` with precise context → the regression arc
- M3: domain rules in `copilot-instructions.md` so Copilot never ignores CET regimes
- M4: writing the failing summer test before fixing the converter — and building the validator on top
- M5: SODA checks catch structural problems the regression runner wouldn't
- M6: knowing when to verify agent output vs. trust it — confident output is not the same as correct output
