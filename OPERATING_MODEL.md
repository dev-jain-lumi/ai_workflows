# Operating Model

## System Flow

Context -> Workflow -> Artifact -> Decision or Action Log -> Review

## Intake to Delivery

1. Capture demand in BACKLOG.md.
2. Triage each item: ignore, delegate, discuss, decide, plan, execute.
3. Route to one workflow in Workflows/.
4. Generate output from Templates/.
5. Save result under Artifacts/.
6. Run relevant checklist in Evals/.
7. Track follow-ups in decision logs and recurring reviews.

## Decision Taxonomy

- Decision Type: strategic or operational
- Reversibility: reversible or irreversible
- Time Horizon: this week, this quarter, this year
- Decision Rights: owner, approver, informed stakeholders

## Review Cadence

- Daily: backlog processing and attention list
- Weekly: delivery and risk reviews
- Monthly: strategy alignment and decision log cleanup

## Escalation SLA

- P0 blocker: escalate same day
- P1 blocker: escalate within 2 working days
- P2 risk: include in weekly review
